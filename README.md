== Алгоритм наполнения клона на WordPress ==

=== У себя локально ===
# Сотворяем тему при помощи Artisteer, в header.php, footer.php, sidebar-footer.php трём к чертовой матери копирайты артистера - еще их нам тут не хватало (искать по слову Artisteer, естественно).
# Кладём тему в папку тем зеброида, разворачиваем проект локально включая вся плагины
# Для каждого клона логин-пароль администратора должны быть уникальными (чтобы не палиться)
# После разворота идём в '''Внешний вид -> Настроить''' и меняем описание проекта с \"Ещё один сайт на WordPress\" на соответствующее
# '''Записи -> Все записи'''. Валим \"Привет мир\"
# '''Страницы -> Все страницы'''. Валим \"Пример страницы\"
# '''Плагины -> Установленные'''. Вырубаем Hello Dolly
# '''Пользователи -> Все пользователи'''. Создаём пользователей для RSS-граббера и импортёра XML файлов. Можно несколько. Тип пользователя - участник. Пароль от балды.
# '''Инструменты -> Импорт -> WordPress'''. Включаем и активируем плагин - нужен будет для заливки зеброй.
# В случае использования WRP внести в style.css темы следующий код (в самый низ):<source lang=\"css\">.feednewscontent img.news-category {
  float: left;
  margin-right: 10px;
  margin-bottom: 10px;
}</source> Эта штука поможет картинкам новостей грабленных по RSS отображаться как надо
# '''Параметры -> Постоянные ссылки.''' Переключаем в '''\"Месяц и название\"'''
# '''Параметры -> Обсуждение''' делаем вот так как на картинке справа ----->  [[Image:2013-09-02_0823.png|thumb|Параметры -> Обсуждение]]
# В корне сайта создаём файл robots.txt следующего содержания:<source lang=\"txt\">User-agent: *
Disallow: /wp-admin/
Disallow: /wp-includes/

Host: currentclone.ru
Sitemap: http://currentclone.ru/sitemap.xml</source> Само собой currentclone.ru заменяем на нужный нам домен (боевой, production, не клона, то есть не site.com.clone, a site.com)
# '''Плагины -> Установленные'''. Вырубаем '''SI CAPTCHA Anti-Spam''' - '''Settings''' делаем вот так ----------->  [[Image:http://content.screencast.com/users/GeXu3/folders/Jing/media/a006395b-ce5e-4905-b6ff-f20c302ee5bc/2013-09-27_0339.png|thumb|SI CAPTCHA Anti-Spam -> Settings]]
# '''Управление пользователями''' - создаём еще 4-5 пользователей с разными никами. Тип - обычный зарегистрированный. Пароли от балды. От них будем новости писать.

=== Локальная заливка контента. ===
# Контент для зеброида обязательно СТАТЕЙНЫЙ. Не новостной.
# Контент компонуется по соответствующим категориям в зеброиде, наполняется картинками, чистится от лишних тегов, проставляется метками и тегами MORE, расписываем даты публикации.
# Экспортируется контент не одним большим файлом, а несколькими сааамыми маленькими (там вроде от 0.5 метра), каждый следующий файл необходимо экспортировать от следующего пользователя по кругу.

=== Перед заливкой на хостинг. Заливка. ===
# Поискать по дампу базы данных все упоминания локального домена, на котором велось заполнение. У меня, например, это allvidsport.info.clone. Вот так и искать, без http. Также поискать эти упоминания в файлах сайта (total commander это умеет прекрасно делать). Все упоминания локального домена заменить на боевой домен (\"Заменой\" в текстовом редакторе. В моём Notepad++ можно воспользоваться Ctrl+R)
# Грохнуть sitemap.xml и sitemap.xml.gz с корня сайта. Создадим новые когда оно уже будет на своём месте.
# Дамп базы данных положить в /_dumper/backup/ проекта.
# Не забыть поменять пароль к базе данных в '''wp-config.php'''
# Упаковать всё это дело в '''ZIP''' архив
# Залить на хостинг, распаковать с панели
# Прямо из панели установить права доступа на папку wp-content '''777''', секюрность в жопу - это клоны а не боевые сайты.
# Идём в http://currentclone.ru/_dumper/ , вбиваем логин-пароль от базы, разворачиваем базу данных из дампа.
# Проверяем работает ли всё. Если да - ура! Если нет - ищем где накосячили.
